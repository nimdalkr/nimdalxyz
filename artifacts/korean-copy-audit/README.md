# 한국어 카피 감사

- 검토일: 2026-07-19
- 범위: 한국어 홈, 프로젝트 9개, 경력 사례 6개, 블로그 3편, 내비게이션과 접근성 라벨
- 원칙: 영어 카피와 사실·수치·URL은 유지하고, 한국어 표현만 다듬는다.

## 참고한 국내 포트폴리오

- [이유진 프로덕트 디자이너 포트폴리오](https://yujiinlee.github.io/)
- [유경아 프론트엔드 포트폴리오](https://kyungah-portfolio.vercel.app/)
- [이주환 프론트엔드 포트폴리오](https://juhwanlog.vercel.app/)
- [이소림 프론트엔드 포트폴리오](https://solim-portfolio.vercel.app/)
- [서범석 프론트엔드 포트폴리오](https://beomxtone.vercel.app/)

문장을 가져오지 않고 다음과 같은 공통점만 참고했다.

- 첫 문장에서 누구이며 무엇을 하는지 바로 밝힌다.
- 추상적인 슬로건보다 만든 것, 맡은 일, 결과를 짧게 쓴다.
- 프로젝트는 문제, 선택, 구현, 확인 가능한 결과와 부족한 점을 구체적으로 설명한다.
- 자기소개에는 필요한 곳에서 `저는`, `만들었습니다`, `맡았습니다`처럼 주체가 보이는 문장을 쓴다.

## 이번 수정 기준

- `증거 표면`, `판단을 보존한다`, `운영 루프`처럼 영어 문장을 옮긴 듯한 표현을 걷어냈다.
- `실서비스`, `커리어`, `빌드 로그` 등 불필요한 외래어는 문맥에 따라 `실제 서비스`, `경력`, `제작 기록`으로 바꿨다.
- 프로젝트 상세 순서를 `무엇이 불편했나 → 어떻게 풀었나 → 무엇을 만들었나 → 확인할 수 있는 결과 → 아직 부족한 점 → 다음에 할 일`로 정리했다.
- 확인할 수 없는 수치는 `포트폴리오 기재 수치`로 명시했다.
